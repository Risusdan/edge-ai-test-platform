# Dig Deeper

Explain the underlying concepts behind what the user just encountered, going one or two layers deeper than the surface-level usage.

## Process

1. Identify the concept(s) the user is working with right now
2. Ask: "Which of these do you want to dig into?" if multiple concepts are in play — or proceed directly if there's an obvious one
3. Explain using this structure:
   - **What it is** — one-sentence definition
   - **Why it exists** — what problem does it solve? What was life like before it?
   - **How it works** — simplified mental model (analogy encouraged)
   - **In our project** — connect back to how it's used here
   - **Going further** (optional) — one link or keyword for self-study if they want even more depth

## Rules

- No jargon without explanation
- Use analogies from everyday life when possible
- If the user's current understanding is wrong, correct gently — say what's right, not just what's wrong
- Keep each explanation focused — one concept at a time, not a textbook chapter
